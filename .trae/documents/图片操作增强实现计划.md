# 图片操作增强实现计划

## 📋 计划概述

### 目标
增强图片转Base64和Base64转图片操作的功能，实现：
1. 图片转Base64操作支持从用户选中的图片中读取并转换
2. 添加输出格式单选框控制输出格式（Data URL/纯Base64/HTML标签）
3. Base64转图片操作添加输出格式单选框（preview/file）

### 成功标准
- 用户可以选中图片文件，图片转Base64操作能直接处理
- 两个操作都有清晰的输出格式单选框控制
- 配置UI显示在操作卡片中，与现有配置保持一致
- 所有功能都能正常工作且向后兼容

### 受众
- 需要频繁处理图片Base64转换的用户
- 希望有更多格式选择权的用户
- 需要图片文件保存功能的用户

### 范围
- **包含**：文件读取扩展、UI配置增强、格式输出优化
- **排除**：复杂的图片编辑功能、批量处理、云存储集成

### 阶段一：扩展文件选择功能

#### 1.1 修改selectFileAsInput方法
**文件**: `src/codec-view.ts`

**修改策略**：
```typescript
private async selectFileAsInput(): Promise<void> {
    const input = document.createElement('input');
    input.type = 'file';
    input.accept = '*'; // 支持所有文件类型
    
    input.onchange = async (event) => {
        const file = (event.target as HTMLInputElement).files?.[0];
        if (!file) return;

        try {
            let fileContent: string;

            // 对于图片文件，提供实际数据供操作使用
            if (file.type.startsWith('image/')) {
                // 读取图片文件并转换为Data URL
                const arrayBuffer = await file.arrayBuffer();
                const bytes = new Uint8Array(arrayBuffer);
                const base64 = this.arrayBufferToBase64(bytes);
                const mimeType = file.type;
                fileContent = `data:${mimeType};base64,${base64}`;
                
                // 存储文件信息供后续操作使用
                (this as any).selectedImageFile = {
                    name: file.name,
                    type: file.type,
                    size: file.size,
                    dataUrl: fileContent
                };
            } else {
                // 其他文件：显示文件信息
                fileContent = `文件名称: ${file.name}\n文件类型: ${file.type}\n文件大小: ${this.formatFileSize(file.size)}`;
            }

            // 将文件内容放入输入框
            const inputArea = this.containerEl.querySelector('.codec-input-area') as HTMLTextAreaElement;
            if (inputArea) {
                inputArea.value = fileContent;
                inputArea.dispatchEvent(new Event('input', { bubbles: true }));
                new Notice(`已选中文件: ${file.name}`);
            }
        } catch (error) {
            new Notice(`文件选择失败: ${error instanceof Error ? error.message : '未知错误'}`);
        }
    };

    input.click();
}
```

### 阶段三：修改图片转Base64操作逻辑

#### 3.1 更新操作执行逻辑
**修改文件**: `src/operations/implementations/images/image-to-base64.ts`

```typescript
protected async executeLogic(input: string, config: Record<string, unknown>): Promise<string> {
    try {
        const format = (config.format as string) || 'data-url';
        const includeMime = (config.includeMime as boolean) !== false;

        // 处理文件信息格式 - 支持从文件中读取
        if (input.startsWith('文件名称:') && input.includes('文件类型:')) {
            // 检查是否是图片文件
            if (input.includes('image/') || input.includes('image/png') || 
                input.includes('image/jpeg') || input.includes('image/gif')) {
                
                throw new Error('检测到图片文件信息。\n\n图片转Base64操作可以处理图片文件。\n\n请确保：\n1. 使用"选中"按钮选择了图片文件\n2. 图片会自动转换为Data URL格式\n3. 再次执行此操作进行格式转换\n\n如果仍然看到此错误，请重新选择图片文件。');
            } else {
                throw new Error('此操作仅支持图片文件。检测到文件类型不是图片。');
            }
        }

        // 如果输入是Data URL格式，提取Base64部分
        if (input.startsWith('data:image/')) {
            const parts = input.split(',');
            if (parts.length >= 2) {
                const base64Part = parts[1];
                return this.formatBase64Output(base64Part, format, includeMime);
            } else {
                throw new Error('无效的Data URL格式');
            }
        }

        // 验证是否是纯Base64图片数据
        this.validateImageBase64(input);
        
        // 处理纯Base64数据
        return this.formatBase64Output(input, format, includeMime);
    } catch (error) {
        throw new Error(`图片Base64转换失败: ${error instanceof Error ? error.message : '未知错误'}`);
    }
}
```

```typescript
protected async executeLogic(input: string, config: Record<string, unknown>): Promise<string> {
    try {
        const outputFormat = (config.outputFormat as string) || 'preview';

        // 清理输入数据
        const cleanedInput = this.cleanBase64Input(input);
        
        // 验证Base64图片数据
        this.validateImageBase64(cleanedInput);

        switch (outputFormat) {
            case 'preview':
                return this.generateImagePreview(cleanedInput);
            case 'file':
                return await this.saveAsImageFile(cleanedInput, config);
            case 'data-url':
                return this.generateDataURL(cleanedInput);
            default:
                return cleanedInput;
        }
    } catch (error) {
        throw new Error(`Base64转图片失败: ${error instanceof Error ? error.message : '未知错误'}`);
    }
}
```

### 文件处理流程

#### 图片文件处理流程
```
用户选择图片文件
    ↓
selectFileAsInput() 检测图片类型
    ↓
读取文件 → ArrayBuffer → Uint8Array
    ↓
转换为Base64 → 添加MIME前缀
    ↓
生成Data URL: data:image/png;base64,...
    ↓
放入输入框 + 存储文件信息
    ↓
图片转Base64操作处理
    ↓
根据配置输出格式转换
```

## 📝 关键技术决策

### 1. 文件读取方式选择
**决策**：扩展selectFileAsInput方法而不是创建新的图片选择器
**理由**：
- 保持UI简洁性
- 与现有文件处理流程一致
- 减少用户的学习成本

### 2. 配置UI设计模式
**决策**：使用单选框而不是下拉选择器
**理由**：
- 更直观的用户体验
- 与现有配置UI风格一致
- 选项数量少（2-3个），单选框更合适

### 3. 图片存储策略
**决策**：在CodecView中存储选中图片的信息
**理由**：
- 避免重复读取文件
- 支持多个操作使用同一个文件
- 便于错误处理和调试

### 4. HTML渲染限制处理
**决策**：保持textarea输出，不修改为HTML渲染
**理由**：
- 避免破坏现有功能
- 防止XSS安全风险
- 保持插件的文本处理核心定位
